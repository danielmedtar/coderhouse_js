Scope - Saludar

Ejercicio 1. Declaración y llamado de la función

Si la funcion encuentra una variable en su ámbito local, aunque tenga el mismo nombre de una en el ámbito global global, va a usar siempre la del ámbito local. Siempre busca primero en su propio ámbito antes de ejecutar.

En caso que en dos funciones distintas hayan variables con el mismo nombre, estas no se van a superponer ni confundir entre sí, ya que las variables declaradas dentro de funciones, solo viven dentro de ellas mismas.

Ejercicio 2. Parámetros

Un parámetro básicamente es una variable local para la función. Al momento de llamar a una función con parámetros, dentro de los paréntesis debe colocarse el valor que queremos que tome para ejecutarse. En el caso del ejemplo: saludarDeNuevo(saludo), pero puede colocarsele cualquier cosa.

Ejercicio 3. Llamar funciones dentro de otras funciones. Ver ejemplo 3 en archivo app.js

Para que esto se ejecute correctamente, dentro de una función solo se hace el llamado a la otra, es decir, el saludarDeNuevo("Mensaje cualquiera"), más no se declara esa funcion dentro de otra.

Toda función se declara dentro de su propio espacio.

Ejercicio 4. Sumar números

Si necesito que la función devuelva algo, como por ejemplo, el resultado de la suma para poder mostrarlo en consola, o usarlo posteriormente, es necesario indicarle a la función que DEBE devolver algo, para eso se usa la palabra reservada return. Ver ejemplo

Es mas recomendable escribir una sola función para cada tarea, y no una función que haga varias tareas. Siempre definirlas con nombres lo mas descriptivos posible. Todo esto para mejor legibilidad y entendimiento

La palabra reservada return devuelve valores, pero también corta un ciclo. Ver ejemplo del Préstamo.
